# Launch Checklist and Week-1 Plan

## Pre-Launch Checklist
1. Confirm Pages deployment from `main`.
2. Run generator scripts and commit outputs:
   - `types_100.json`
   - `data/cards/*`
   - `u/*/index.html`
3. Validate workflows are enabled:
   - `build-pages`
   - `generate-data-packs`
   - `generate-readme-cards`
   - `update-hall-of-fame`
4. Confirm `README.md` includes embed snippet and project pitch.
5. Confirm Hall of Fame issue template is available.
6. Validate demo profile flow on production URL.

## Launch Day
1. Create release tag `v0.1.0`.
2. Publish launch post with:
   - product URL
   - one share card example
   - one README embed example
3. Pin issue for user feedback.
4. Monitor issues for API edge-case errors.

## Week-1 Shipping Plan
1. Day 1: launch + feedback collection.
2. Day 2: fix top 3 onboarding friction points.
3. Day 3: tune centroid/threshold issues reported by users.
4. Day 4: improve card visuals and codex readability.
5. Day 5: publish first "community Hall of Fame" update.
6. Day 6-7: ship one visible improvement and post changelog.

## Post-Launch Metrics (Week 1)
1. Total analyses completed.
2. Share card downloads.
3. README snippet copies.
4. Hall of Fame submissions.
5. Net stars gained.
